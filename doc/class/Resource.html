Class: Resource

Defined in: src/fire-resource.coffee

Overview

Note: Resource classes are created by the FireResource factory

Base resource class

Below the different way to use the FireResource factory

Examples:

Basic model 

app.module('myApp').factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users')

Class methods not prefixed with `$` are chainable 

factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users')
    .hasMany 'conversations'
    .hasOne 'displayedConversation',
      className: 'Conversation'
      inverseOf: false

Pass a callback as the last argument to access the resource from within 

factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users'), ->
    @hasMany 'conversations'
    @hasOne 'displayedConversation',
      className: 'Conversation'
      inverseOf: false
    @::myCostomInstanceMethod = -> "i'm custom"

Parameters:

  • ref ( ref ) — firebase reference
  • opts ( Object ) — options (optional)
  • callack ( Function ) — optional function called in the context of the defined Resource (to add methods, relations, or override stuff)

Returns:

  • ( Resource ) — the Resource class ready to be used by the angular factory

. (Resource) hasMany(name, opts = {}, cb)

One to Many association builder

Examples:

factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users'), ->
    @hasMany 'conversations'

the above association gives you access to the association collection via a getter method prefixed by `$` : 

user.$conversations()

you can customize its ref any time by passing it an init function 

user.$conversations (baseRef, init) -> init(new Firebase.util.Scroll(baseRef, '$key')).$next(10)

the association collection is cached on the parent instance on a property prefixed by `$$` 

user.$conversations() === user.$$conversations

when the parent user is `$destroyed`, its association collection are too, but not its associated instances 

user.$destroy()
user.$$conversations === undefined

Parameters:

  • association ( String ) — name
  • association ( Object ) — option
  • callback ( Function ) — function customize the association collection ref or add some method

Options Hash: (options):

  • className ( String ) — targetted resource factory name, defaults to name.camelize(true)
  • inverseOf ( String ) — inverse association, defaults to Resource.$name, if false association will be considered one sided

Returns:

  • ( Resource ) — the current resource

. (Resource) hasOne(name, opts = {})

One to One association builder

Examples:

factory 'Message', (FireResource, $firebase) ->
  FireResource $firebase.child('messages'), ->
    @hasOne 'user', inverseOf: false

  is the same than doing :
  FireResource $firebase.child('messages'), ->
    @hasOne 'user', className: 'User', forereignKey: 'userId', inverseOf: false

the above association dynamically create the following methods on any Message instance 

message.$user() #get the associated user
message.$setUser( userInstance ) #associate a user instance
message.$createUser( userData ) #create an associated user instance

Parameters:

  • name ( String )
  • options ( Object )

Options Hash: (options):

  • className ( String ) — targetted resource factory name, defaults to name.camelize(true)
  • foreignKey ( String ) — property used to store the associated instance key, defaults to name + 'Id'
  • inverseOf ( String ) — inverse association, defaults to Resource.$name, if false association will be considered one sided

Returns:

  • ( Resource ) — the current resource
