Class: Resource

Defined in: src/fire-resource.coffee

Overview

Note: Resource classes are created by the FireResource factory

Base resource class

Below the different way to use the FireResource factory

Examples:

Basic model 

app.module('myApp').factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users')

Chaining class method calls 

factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users')
    .hasMany 'conversations'
    .hasOne 'displayedConversation',
      className: 'Conversation'
      inverseOf: false

Accessing the resource from within 

factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users'), ->
    @hasMany 'conversations'
    @hasOne 'displayedConversation',
      className: 'Conversation'
      inverseOf: false
    @::myCostomInstanceMethod = -> "i'm custom"

Parameters:

  • resourceRef ( ref ) — firebase reference
  • resourceOptions ( Object ) — options (optional)
  • callack ( Function ) — optional function called in the context of the defined Resource (to add methods, relations, or override stuff)

. (Resource) hasMany(name, opts = {}, cb)

One to Many association builder

Examples:

factory 'User', (FireResource, $firebase) ->
  FireResource $firebase.child('users'), ->
    @hasMany 'conversations'

Parameters:

  • association ( String ) — name
  • association ( Object ) — option
  • callback ( Function ) — function customize the association collection ref or add some method

Options Hash: (options):

  • className ( String ) — targetted resource factory name, defaults to name.camelize(true)
  • inverseOf ( String ) — inverse association, defaults to Resource.$name, if false association will be considered one sided

Returns:

  • ( Resource ) — the current resource

. (Resource) hasOne(name, opts = {})

One to One association builder

Parameters:

  • name ( String )
  • options ( Object )

Options Hash: (options):

  • className ( String ) — targetted resource factory name, defaults to name.camelize(true)
  • foreignKey ( String ) — property used to store the associated instance key, defaults to name + 'Id'
  • inverseOf ( String ) — inverse association, defaults to Resource.$name, if false association will be considered one sided

Returns:

  • ( Resource ) — the current resource
